### Constraints:
- Your solution must be compatible with PyTorch.
- You can use `torch.utils.cpp_extension.load_inline` to compile and load custom CUDA kernels.
- You should aim to optimize at least one of the operations in the forward pass.
- You are encouraged to combine multiple operations into a single kernel where possible.

### Example of how to use `load_inline`: 

```python
from torch.utils.cpp_extension import load_inline

custom_op_source = """
#include <torch/extension.h>
#include <cuda_runtime.h>

__global__ void my_custom_op_kernel(...) {
    // Custom CUDA kernel implementation
}

torch::Tensor my_custom_op_cuda(...) {
    // Launching the custom kernel and returning the result
}
"""

custom_op_cpp_source = (
    "torch::Tensor my_custom_op_cuda(...);"
)

custom_op = load_inline(
    name="my_custom_op",
    cpp_sources=custom_op_cpp_source,
    cuda_sources=custom_op_source,
    functions=["my_custom_op_cuda"],
    verbose=True,
    extra_cflags=[""],
    extra_ldflags=[""],
)
```

```python
class MyModel(nn.Module):
    def __init__(self):
        super(MyModel, self).__init__()
        self.custom_op = custom_op

    def forward(self, x):
        x = self.custom_op.my_custom_op_cuda(x)
        return x
```

```python
model = MyModel().cuda()
input_tensor = torch.randn(32, 128).cuda()
output_tensor = model(input_tensor)
print(output_tensor.shape)
```
